# Formatting and Style Standards

## Markdown Formatting

- Use consistent heading levels
- Ensure code blocks have proper language tags
- Use consistent table formatting
- Maintain consistent list formatting

## Code Blocks

- Always specify language for syntax highlighting
- Use appropriate code block types (```language vs ```)
- Ensure indentation is consistent
- Include file paths when referencing code files

## Tables

- Use consistent table formatting
- Ensure tables are properly aligned
- Use markdown table syntax consistently
- Add table captions where helpful

## Links and References

- **Follow citation guidelines**: See `content/how-to-cite.md` for exact format
- Use consistent reference format: `Author(s) (Year) [Title](link), Venue`
- Ensure internal links use relative paths (e.g., `[link](0-1-intro-book.md)`)
- Verify external links are accessible
- Use descriptive link text
- All references must be cited in text; don't list uncited references
- Use inline clickable links for further reading (per `how-to-contribute.md`)

## Mathematical Notation

- Use LaTeX math notation consistently
- Ensure math blocks are properly formatted ($$ vs $)
- Use consistent variable naming
- Verify mathematical expressions render correctly

## File Organization

- Maintain consistent file naming conventions
- Ensure directory structure is logical
- Keep related files together
- Use appropriate file extensions
